What is TLED?

TLED is a Matter-over-Thread LED controller for ESP32-C6. It lets you control addressable LED strips (WS2812B, SK6812, etc.) through your smart home using the Matter protocol over Thread networking.

Key features:

  • Works with Home Assistant, Apple Home, Google Home, and other Matter controllers
  • Thread mesh networking - no WiFi needed, low power, self-healing network
  • Full RGB color control with smooth transitions
  • Configurable via USB serial - no recompiling needed

Requirements

  • ESP32-C6 board - any board with USB-C (e.g., DFRobot Beetle ESP32-C6)
  • Addressable LED strip - WS2812B, WS2811, or SK6812
  • 5V power supply - sized for your LED count (~60mA per LED at full white)
  • Thread border router - HomePod Mini, Apple TV 4K, or dedicated router like SLZB-06
  • Matter controller - Home Assistant with Matter integration, Apple Home, etc.

Installation Steps

  1. Connect your ESP32-C6 via USB to your computer
  2. Click "Install TLED Firmware" in the Install tab
  3. If prompted, hold the BOOT button while clicking Install
  4. Wait for the flash to complete
  5. Go to the Configure tab and connect to the device
  6. Set your LED count, GPIO pin, and other settings
  7. Click Save & Reboot
  8. Scan the QR code with your smart home app to commission

Configuration Options

Number of LEDs - How many LEDs in your strip (1-1000)

GPIO Pin - Data pin connected to your LED strip. Common choices:

  • GPIO 5 (default) - works on most ESP32-C6 boards
  • Check your board's pinout for available GPIOs

LED Type

  • WS2812B - Most common, 800kHz, GRB order
  • WS2811 - Older, 400kHz, often RGB order
  • SK6812 - Similar to WS2812B, some have RGBW

RGB Order - Color byte order. If colors look wrong, try a different order:

  • GRB - Most WS2812B strips
  • RGB - Some WS2811 and older strips
  • BGR/RBG - Less common, try if colors are swapped

Max Brightness - Limits maximum brightness (1-255). Lower values save power and reduce heat.

Serial Commands

help - Show available commands

config - Show current configuration

set leds <n> - Set number of LEDs

set gpio <n> - Set GPIO pin

set type <ws2812b|ws2811|sk6812> - Set LED type

set order <grb|rgb|bgr|rbg> - Set RGB order

set brightness <1-255> - Set max brightness

save - Save configuration to flash

reboot - Restart the device

factory - Factory reset (erases all settings and commissioning)

Troubleshooting

Can't flash / "No port selected"

  • Make sure you're using Chrome or Edge (Firefox doesn't support Web Serial)
  • Try holding the BOOT button while clicking Install
  • Check USB cable - some cables are charge-only without data lines

Device boot loops / "No bootable app partitions"

  • Flash failed mid-way. Try flashing again with a fresh install.

Can't find device during commissioning

  • Commission within 30 seconds of boot (BLE advertising slows down after that)
  • If it says "Fabric already commissioned", run factory command
  • Make sure your Thread border router is nearby and working

Wrong colors on LED strip

  • Try different RGB Order settings (GRB, RGB, BGR, RBG)

LEDs don't light up

  • Check power supply - LEDs need 5V, not 3.3V
  • Verify GPIO pin matches your wiring
  • Make sure LED count is correct
